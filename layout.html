Aligning to a Flexible Grid

November 5, 2011

  • writing
  • design
  • software
  • books

The main Lab page and the Typography Page are fixed width, aligned to a 960px grid. This page is laid out on a flexible grid, thanks to Ethan Marcotte's excellent book Responsive Web Design. The grid itself (and defined by the #wrap element) is 960 pixels wide. The grid consists of 12 columns of 58px each, with 24px gutters. To see the grid, click the button in the upper right corner of the browser window. Another tool I’ve used is a bookmarklet from Javascript.org that I keep in my bookmark bar in Safari. The former is good in that users can see it and also has an option—currently disabled—to display horizontal lines so that the layout grid and the vertical scale can be displayed at the same time. The latter is less obtrusive and good for development work.

The width of the content and sidebar elements in pixels is determined by how many columns and gutters each takes up. These pixel widths are then converted to percentages so that when the browser window is narrowed below 960px, the proportions are maintained. The center content is floated left, but has a left margin equal to 3 columns and three gutters. The article dates, tags and sidenotes are then floated left as well, but with negative left margins. The math here can get out of hand, so I use an Excel spreadsheet that does it all for me. I set the parameters of the grid, and the desired width of elements and it spits out the necessary widths which I put in the CSS.

And yes, typographically this page isn’t much to look at—that’s what the Typography page is for—but it makes the CSS easier for me to tweak. Eventually the best of both pages will merge.

That should be a flexible image. It should resize as the text column is narrowed. No width or height is defined in the HTML and the img element is given a width of 100% in CSS.

Next Stop: Media Queries

January 9, 2012

  • rants
  • politics

The next step here will be to implement media queries so if the browser window is narrowed, or for use on small viewport devices like iPhones, the various divs change their properties to account for the smaller overall width.

The first “width trigger” is 680 pixels. This is due to changing the underlying grid from 12 columns to 8. Eight columns x 54, plus 7 gutters x 24 = 632. Include 1 gutter of 24 pixels on each side as margins and you get 680. Key changes here:

  1. The nav menu at the top initially has a width of 32% so that the four items each occupy the four right most columns. As the overall browser window narrows, these get cramped and hence, the width changes to 50%, which pretty closely aligns them with the four right columns in our now eight column grid.
  2. The larger left margin of the main content column is removed and then the meta and sidenote containers have their left floats and negative left margins removed bring them back into the normal flow.
  3. The four paragraphs in the footer change to a 2 by 2 grid.

The other trigger occurs at 380 pixels. All the floats are removed and all elements are set to the full width of the browser.
